# Veritas

**Proof of authenticity for photos in an AI-generated world.**

Veritas is a cryptographic image verification system that proves whether a photo was taken by a real camera or generated by AI. Instead of trying to detect AI images (which gets harder as AI improves), we flip the problem: real photos get cryptographically signed by trusted hardware, and anything without a valid signature is considered unverified.

## The Problem

AI can now generate photorealistic images that are indistinguishable from real photos. Traditional detection methods are an arms race—they fail as AI models improve. We need a way to prove authenticity, not detect forgery.

## Our Solution

Veritas uses hardware-backed cryptography to create unforgeable proof that an image came from a real camera:

1. **Capture**: Camera takes photo → hash the pixel data → TPM signs the hash → embed signature in image
2. **Verify**: Load image → recompute hash → check signature → confirm authenticity

If the signature is valid, the photo is real. If not, it could be AI-generated or tampered with.

## How It Works

### Capture App

When you take a photo with Veritas:
- Webcam captures the image
- System generates a SHA-256 hash of the exact pixel data
- Your computer's TPM (Trusted Platform Module) signs that hash with a private key that can't be extracted
- The signature, hash, and device certificate are embedded in the image file

### Verification App

When someone wants to verify an image:
- Load the image and extract embedded metadata
- Recalculate the hash from current pixel data
- Verify the signature using the device's public key
- If the hashes match and signature is valid → verified ✓
- If anything is wrong → unverified ✗

### Why This Works

- **AI images can't pass**: They don't have signatures from real cameras
- **Tampered photos fail**: Changing even one pixel breaks the signature
- **Can't be faked**: TPM won't let anyone steal or copy the signing key
- **Future-proof**: Unlike AI detection, this doesn't break as AI improves

## Security Model

**Assumptions:**
- Private signing key is protected in TPM hardware
- Images can only be signed during trusted capture
- Modifying the image invalidates the signature

**Attack Resistance:**
- Copying metadata to another image won't work (signature is bound to pixel data)
- Screenshots of verified images won't verify (new capture, no signature)
- Editing a verified image breaks the hash and fails verification

## Why Not Just Use AI Detection?

AI detection tools are probabilistic and get worse as AI improves. Veritas provides mathematical certainty through cryptography. It's like the difference between "this signature looks fake" vs. "this signature is cryptographically invalid."
